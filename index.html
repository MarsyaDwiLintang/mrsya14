Pengantar Extension Methods

Extension Methods adalah fitur pada Dart yang memungkinkan programmer menambahkan fungsi baru ke tipe data atau class yang sudah ada tanpa perlu mengubah soruce code aslinya. Dengan fitur ini, pengguna method menjadi lebih praktis, singkat, dan mudah dibaca.

01. Masalah tanpa Extension Methods

Tanpa Extencion, method harus dipanggil melalui utility class sehingga kode terlihat lebih panjang

DART SOURCE CODE

// Nama  : Marsya Dwi Lintang
// NIM   : 251410028
// Kelas : SI2A

class StringUtils {
  static String capitalize(String text) {
    if (text.isEmpty) return text;
    return text[0].toUpperCase() + text.substring(1);
  }

  static bool isEmail(String text) {
    return text.contains('@');
  }
}

void main() {
  // Penggunaan kurang natural
  var name = 'john';
  var capitalized = StringUtils.capitalize(name);
  var isEmail = StringUtils.isEmail('test@mail.com');

  // Cetak variabel
  print('Original name: $name');
  print('Capitalized name: $capitalized');
  print('Is "test@mail.com" an email? $isEmail');
}

02. Solusi dengan Extension Methods

Dengan extension, method dapat dipanggil langsung dari object sehingga lebih praktis dan mudah dibaca.

DART SOURCE CODE

// Nama  : Marsya Dwi Lintang
// NIM   : 251410028

extension StringExtensions on String {
  String capitalize() {
    if (isEmpty) return this;
    return this[0].toUpperCase() + substring(1);
  }

  bool get isEmail => contains('@');

  String reverse() {
    return split('').reversed.join();
  }
}

void main() {
  // Penggunaan lebih natural
  var name = 'john';
  var capitalized = name.capitalize();
  var isEmail = 'test@mail.com'.isEmail;
  var reversed = 'hello'.reverse();

  // Cetak variabel
  print('Original name: $name');
  print('Capitalized name: $capitalized');
  print('Is "test@mail.com" an email? $isEmail');
  print('Reversed "hello": $reversed');
}

03. Sintaks Extension Methods

Sintaks Extension Methods digunakan untuk menambahkan method baru ke tipe data tertentu menggunakan keyword extension dan on. Di dalam extension dapat dibuat method, getter, setter, maupun operator agar tipe data memiliki fungsi tambahan tanpa mengubah class aslinya.

DART SOURCE CODE

// Nama  : Marsya Dwi Lintang
// NIM   : 251410028
// Kelas : SI2A

extension DateTimeExtensions on DateTime {

  // Method: format tanggal
  String toFormattedString() {
    return '$day/$month/$year';
  }

  // Getter: cek apakah hari ini
  bool get isToday {
    var now = DateTime.now();

    return year == now.year &&
           month == now.month &&
           day == now.day;
  }

  // Operator: tambah durasi
  DateTime operator +(Duration duration) {
    return add(duration);
  }
}

void main() {

  var today = DateTime.now();
  var oldDate = DateTime(2026, 5, 23);
  var nextMonth = today + Duration(days: 30);

  print('Tanggal hari ini: ${today.toFormattedString()}');
  print('Tanggal lama: ${oldDate.toFormattedString()}');
  print('Apakah today adalah hari ini? ${today.isToday}');
  print('Apakah oldDate adalah hari ini? ${oldDate.isToday}');
  print('30 hari berikutnya: ${nextMonth.toFormattedString()}');
}

05. Extension untuk Null Safety

Extension untuk Null Safety digunakan untuk membantu menangani data nullable seperti String? atau int? agar lebih aman saat digunakan. Dengan extension ini, programmer dapat menghindari error null dan membuat kode menjadi lebih singkat, rapi, serta mudah dibaca.

DART SOURCE CODE

// Nama  : Marsya Dwi Lintang
// NIM   : 251410028
// Kelas : SI2A

extension NullableStringExtensions on String? {
  // Safe access untuk nullable string
  String orEmpty() => this ?? '';

  bool get isNullOrEmpty =>
      this == null || this!.isEmpty;

  bool get isNotNullOrEmpty =>
      !isNullOrEmpty;

  String withDefault(String defaultValue) =>
      this ?? defaultValue;
}

void main() {
  // Penggunaan
  String? name;

  print(name.orEmpty());              // ''
  print(name.isNullOrEmpty);          // true
  print(name.isNotNullOrEmpty);       // false
  print(name.withDefault('Guest'));   // Guest
}

06. Praktik 1- Utilities untuk Dart Types

Extension pada Dart juga dapat digunakan untuk membuat utility tambahan pada tipe data tertentu, misalnya pada tipe angka (num). Dengan cara ini, programmer dapat menambahkan fungsi seperti format angka, pengecekan range, atau operasi matematika agar penggunaan kode menjadi lebih praktis dan reusable.

DART SOURCE CODE

// Nama  : Marsya Dwi Lintang
// NIM   : 251410028
// Kelas : SI2A

extension NumberExtensions on num {
  String toRupiah() {
    return 'Rp${toStringAsFixed(0).replaceAllMapped(
      RegExp(r'(\d{1,3})(?=(\d{3})+(?!\d))'),
      (match) => '${match[1]}.',
    )}';
  }

  bool isBetween(num min, num max) {
    return this >= min && this <= max;
  }

  num power(int exponent) {
    var result = this;

    for (var i = 1; i < exponent; i++) {
      result *= this;
    }

    return result;
  }
}

void main() {
  var price = 1500000;
  print(price.toRupiah());        // Rp1.500.000
  print(5.isBetween(1, 10));      // true
  print(2.power(3));              // 8
}

07. Generic Extensions

Generic Extensions digunakan agar extension dapat bekerja pada berbagai tipe data tanpa perlu membuat extension baru untuk setiap tipe. Dengan generic, extension menjadi lebih fleksibel, reusable, dan dapat digunakan pada banyak jenis data seperti List, List, dan lainnya.

DART SOURCE CODE

// Nama  : Marsya Dwi Lintang
// NIM   : 251410028
// Kelas : SI2A

extension ListExtensions on List {
  T? getOrNull(int index) {
    if (index < 0 || index >= length) {
      return null;
    }

    return this[index];
  }

  List> chunk(int size) {
    var chunks = >[];

    for (var i = 0; i < length; i += size) {
      chunks.add(
        sublist(
          i,
          i + size > length
              ? length
              : i + size,
        ),
      );
    }

    return chunks;
  }

  List distinct() {
    var result = [];

    for (var item in this) {
      if (!result.contains(item)) {
        result.add(item);
      }
    }

    return result;
  }
}

void main() {
  var numbers = [1, 2, 3, 4, 5, 6, 7];
  print(numbers.chunk(3));                // [[1,2,3],[4,5,6],[7]]
  print([1, 2, 2, 3, 3, 3].distinct());   // [1, 2, 3]
  print(numbers.getOrNull(10));
}

09. Konflik Nama Extension

Konflik nama extension terjadi ketika terdapat dua extension dengan nama method yang sama pada tipe data yang sama. Hal ini membuat Dart bingung menentukan method mana yang harus digunakan sehingga menyebabkan error. Untuk mengatasinya dapat menggunakan prefix import atau show/hide.

DART SOURCE CODE

// Nama  : Marsya Dwi Lintang
// NIM   : 251410028
// Kelas : SI2A

extension StringCaseExtensions on String {
  String capitalize() {
    return this[0].toUpperCase() +
        substring(1);
  }
}

extension TextUtilsExtensions on String {
  String capitalize() {
    return toUpperCase();
  }
}

void main() {

  var text = 'hello';

  print(
    StringCaseExtensions(text)
        .capitalize(),
  );

  print(
    TextUtilsExtensions(text)
        .capitalize(),
  );
}

10. Extension untuk Custom Classes

Extension Methods tidak hanya dapat digunakan pada tipe bawaan Dart, tetapi juga bisa diterapkan pada custom class buatan sendiri. Dengan extension, programmer dapat menambahkan fitur baru pada class tanpa harus mengubah isi class aslinya sehingga kode menjadi lebih rapi dan mudah dikembangkan.

DART SOURCE CODE

// Nama  : Marsya Dwi Lintang
// NIM   : 251410028
// Kelas : SI2A

class Product {
  final String name;
  final double price;

  Product(this.name, this.price);
}

extension DoubleRupiahFormatter on double {

  String toRupiah() {

    final intValue = toInt();

    String numberString =
        intValue.toString();

    String formattedString = '';

    int counter = 0;

    for (
      int i = numberString.length - 1;
      i >= 0;
      i--
    ) {

      formattedString =
          numberString[i] +
          formattedString;

      counter++;

      if (
        counter % 3 == 0 &&
        i != 0
      ) {

        formattedString =
            '.' + formattedString;
      }
    }

    return 'Rp$formattedString';
  }
}

extension ProductExtensions on Product {

  Product withDiscount(
    double percentage,
  ) {

    var discountedPrice =
        price *
        (1 - percentage / 100);

    return Product(
      name,
      discountedPrice,
    );
  }

  String get displayInfo =>
      '$name - ${price.toRupiah()}';

  bool get isExpensive =>
      price > 1000000;
}

void main() {

  var laptop =
      Product(
    'Laptop',
    15000000,
  );

  var discounted =
      laptop.withDiscount(10);

  print(
    discounted.displayInfo,
  );

  print(
    laptop.isExpensive,
  );

  var keyboard =
      Product(
    'Keyboard',
    750000,
  );

  print(
    keyboard.displayInfo,
  );

  print(
    keyboard.isExpensive,
  );
}

11. Praktik 2- Validator Extensions

Validator Extension digunakan untuk membantu proses validasi data seperti email, password, nomor telepon, atau angka. Dengan extension, proses pengecekan data menjadi lebih singkat, rapi, dan mudah digunakan langsung pada object.

DART SOURCE CODE

// Nama  : Marsya Dwi Lintang
// NIM   : 251410028

extension StringValidators on String {

  bool get isEmail {
    return contains('@');
  }

  bool get isStrongPassword {
    return length >= 8;
  }
}

extension IntValidators on int {

  bool get isEven {
    return this % 2 == 0;
  }

  bool get isPositive {
    return this > 0;
  }
}

void main() {

  var email = 'test@mail.com';
  var password = 'Password123';
  var number = 10;

  print(email.isEmail);
  print(password.isStrongPassword);
  print(number.isEven);
  print(number.isPositive);
}

Ringkasan & Best Practices

Extension Methods adalah fitur pada Dart yang digunakan untuk menambahkan method baru ke tipe data atau class tanpa mengubah kode aslinya. Fitur ini membuat kode meniadi lebih singkat, rapi, mudah dibaca, dan reusable. Extension juga mendukung generic type, nul| safety, custom class, qetter, setter hingga operator sehingga sangat membantu dalam pengembangan aplikasi Dart

    Best Practices

  • Gunakan nama Extencion yang jelas dan mudah dipahami
  • Pisahkan extencion berdasarkan tipe data
  • Hindari penggunaan extencion secara berlebihan
  • Perhatikan null safety agar kode tetap aman
  • Tambahkan komentar atau dokumentasi jika diperlukan
